# Production Deploy: Vercel + Render + Neon

This is the **recommended production stack** for HaulBrokr:

| Layer | Provider | URL |
|-------|----------|-----|
| Web app | **Vercel** | `https://haulbrokr.com` |
| API | **Render** (Docker) | `https://haulbrokr-api.onrender.com` (proxied via Vercel `/api`) |
| Database | **Neon** | Serverless Postgres |
| Mobile | **EAS** | App Store + Play Store |
| Auth | **Clerk** | — |
| Payments | **Stripe Connect** | — |
| Email | **Resend** | — |

**Why this split:** Vercel is best for the static React app (CDN, custom domain). Render runs the long-lived Express API. Neon gives managed Postgres with a generous free tier and pairs well with serverless-style deploys.

---

## Architecture

```
                    haulbrokr.com (Vercel)
                           │
         ┌─────────────────┼─────────────────┐
         │                 │                 │
    Static SPA        /api/* rewrite    /mobile-payment
         │                 │                 │
         │         haulbrokr-api.onrender.com
         │                 │
         │            Neon Postgres
         │
    Mobile app ──► https://haulbrokr.com/api  (same origin via Vercel)
```

Mobile uses `EXPO_PUBLIC_DOMAIN=haulbrokr.com` so API calls go through Vercel → Render. Web uses Clerk proxy at `/api/__clerk` on the same domain.

## Phase 2 — Neon database (10 min)

1. Neon → **New Project** → name `haulbrokr-prod`
2. Copy the **pooled connection string** (includes `-pooler` in hostname)
3. Ensure it ends with `?sslmode=require`
4. Example:
   ```
   postgres://user:pass@ep-xxx-pooler.us-east-2.aws.neon.tech/haulbrokr?sslmode=require
   ```
5. From your laptop (with `DATABASE_URL` set), push schema:
   ```bash
   export DATABASE_URL="postgres://..."
   pnpm --filter @workspace/db run push
   ```

6. **Seed staff admin accounts** (username/password login at `/admin/login`):
   ```bash
   export DATABASE_URL="postgres://..."
   STAFF_DEFAULT_PASSWORD='YourSecureProductionPassword' \
     pnpm --filter @workspace/api-server run seed-staff
   ```
   Creates: `ceo`, `president`, `cto`, `cfo`, `accounting`, `it`, `programmer` (roles map to admin permissions).
   Change passwords after first login in production (re-run seed only on empty DB, or update rows manually).

7. **Seed permanent Apple App Store review account** (mobile Clerk login):
   ```bash
   export DATABASE_URL="postgres://..."
   export CLERK_SECRET_KEY="sk_live_..."
   pnpm --filter @workspace/api-server run seed-apple-review
   ```
   Creates/updates `apple-review@haulbrokr.com` with username/password `Apple1demo` / `Apple1demo`, verified email, provider profile, fleet, jobs, and notifications. Add credentials to App Store Connect review notes.

## Phase 3 — Render API (20 min)

1. Push repo to GitHub
2. Render → **New** → **Blueprint** → connect repo
3. Render reads `render.yaml` (API-only blueprint)
4. Set these **secret** env vars in the Render dashboard:

| Variable | Value |
|----------|-------|
| `DATABASE_URL` | Neon pooled connection string |
| `CLERK_SECRET_KEY` | From Clerk dashboard |
| `CLERK_PUBLISHABLE_KEY` | From Clerk dashboard |
| `STRIPE_SECRET_KEY` | `sk_live_...` from Stripe |
| `STRIPE_PUBLISHABLE_KEY` | `pk_live_...` from Stripe |
| `STRIPE_WEBHOOK_SECRET` | `whsec_...` from the Stripe webhook endpoint |
| `RESEND_API_KEY` | From Resend |
| `RESEND_FROM_EMAIL` | `noreply@haulbrokr.com` (verified domain) |
| `ADMIN_USER_IDS` | Your Clerk user ID (comma-separated) |
| `R2_ACCOUNT_ID` | Cloudflare account ID |
| `R2_ACCESS_KEY_ID` | R2 access key ID |
| `R2_SECRET_ACCESS_KEY` | R2 secret access key |
| `R2_BUCKET` | R2 bucket name |
| `R2_PUBLIC_URL` | Public R2/custom CDN base URL |
| `PRIVATE_OBJECT_DIR` | `/haulbrokr/private` |
| `PUBLIC_OBJECT_SEARCH_PATHS` | `/haulbrokr/public` |
| `CORS_ALLOWED_ORIGINS` | `https://haulbrokr.com,https://www.haulbrokr.com` |
| `AUTOMATION_KEY` | Optional; set only if `/api/automation/*` jobs are enabled |

`UPLOAD_TOKEN_SECRET`, `TICKET_QR_SECRET`, and `STAFF_AUTH_SECRET` are auto-generated by Render (staff session cookies).

5. **Do not** set `PAYMENTS_MOCK_MODE` in production.
6. Stripe Dashboard → **Developers → Webhooks** → create endpoint:
   - URL: `https://haulbrokr-api.onrender.com/api/webhooks/stripe`
   - Events: `payment_intent.succeeded`, `payment_intent.payment_failed`, `checkout.session.completed`, `account.updated`
   - Copy the signing secret to Render as `STRIPE_WEBHOOK_SECRET`
7. Wait for deploy → verify:
   ```bash
   curl https://haulbrokr-api.onrender.com/api/readyz
   # → {"status":"ok"}
   ```

8. If your Render service name differs from `haulbrokr-api`, update the rewrite URL in `artifacts/haulbrokr/vercel.json`.

## Phase 4 — Vercel web (15 min)

1. Vercel → **Add New Project** → import GitHub repo
2. **Root Directory:** `artifacts/haulbrokr`
3. Vercel auto-detects `vercel.json` — no manual build settings needed
4. **Environment Variables** (Production):

| Variable | Value |
|----------|-------|
| `VITE_CLERK_PUBLISHABLE_KEY` | Same Clerk publishable key |
| `VITE_CLERK_PROXY_URL` | `/api/__clerk` |

5. Deploy
6. **Domains** → add `haulbrokr.com` and `www.haulbrokr.com`
7. Update DNS at your registrar (Vercel shows exact records)

### Verify web + API proxy

```bash
curl https://haulbrokr.com/api/readyz
# → {"status":"ok"}  (proxied to Render)
```

Open `https://haulbrokr.com` → sign in should work via Clerk.

## Phase 5 — Clerk (10 min)

In [Clerk Dashboard](https://dashboard.clerk.com) → your app:

1. **Domains** → add:
   - `haulbrokr.com`
   - `www.haulbrokr.com`
2. **Paths** → ensure sign-in/sign-up paths match (`/sign-in`, `/sign-up`)
3. **Social connections** → enable Google, Apple (for mobile)
4. **Apple Sign-In** → bundle ID: `com.haulbrokr.mobile`
5. Copy **Publishable key** and **Secret key** to Render + Vercel + EAS

## Phase 7 — Mobile EAS (30 min)

```bash
cd artifacts/haulbrokr-mobile
```

Set EAS secrets (or `.env` for local):

| Secret | Value |
|--------|-------|
| `EXPO_PUBLIC_DOMAIN` | `haulbrokr.com` |
| `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk publishable key |
| `EXPO_PUBLIC_CLERK_GOOGLE_WEB_CLIENT_ID` | Google web client ID (Clerk Dashboard) |
| `EXPO_PUBLIC_CLERK_GOOGLE_IOS_CLIENT_ID` | Google iOS client ID (Clerk Dashboard) |
| `EXPO_PUBLIC_CLERK_GOOGLE_IOS_URL_SCHEME` | Reversed iOS client ID URL scheme |
| `GOOGLE_MAPS_API_KEY` | Android Google Maps SDK key |

```bash
eas secret:create --scope project --name EXPO_PUBLIC_DOMAIN --value haulbrokr.com
eas secret:create --scope project --name EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY --value pk_live_...
eas secret:create --scope project --name GOOGLE_MAPS_API_KEY --value AIza...

eas build --platform ios --profile production
eas build --platform android --profile production

# After TestFlight QA:
eas submit --platform ios
eas submit --platform android
```

ASC App ID `6769841431` is in `eas.json`. Android submit uses the `internal` track and expects a Play Console service account JSON at `artifacts/haulbrokr-mobile/google-service-account.json` (gitignored).

- [ ] Staff login at `/admin/login` → `/admin` command center loads
- [ ] `curl https://haulbrokr.com/api/admin/access` without cookie → `isAdmin: false`
- [ ] Customer sign-up → onboarding → post load
- [ ] Provider sign-up → see open load → place bid
- [ ] Customer accepts bid → job created
- [ ] Provider marks job complete
- [ ] Customer pays (Checkout or saved card)
- [ ] Provider Stripe Connect onboarding works
- [ ] Mobile: sign in → jobs tab shows loads
- [ ] Mobile: Account → Payment Method → add card
- [ ] Admin: compliance review + email notification

## Optional: custom API subdomain

Instead of proxying to `haulbrokr-api.onrender.com`, you can:

1. Render → custom domain `api.haulbrokr.com`
2. DNS CNAME → Render
3. Update `vercel.json` rewrite to `https://api.haulbrokr.com/api/:path*`
4. Mobile can use `EXPO_PUBLIC_DOMAIN=api.haulbrokr.com` (direct API, no Vercel proxy)

The default setup (everything through `haulbrokr.com`) is simpler for Clerk cookies and CORS.

## Troubleshooting

| Problem | Fix |
|---------|-----|
| `/api/readyz` 502 on Vercel | Render API not running or database unavailable — check Render logs and Neon connectivity |
| Clerk sign-in fails | Add `haulbrokr.com` to Clerk domains; verify `VITE_CLERK_PROXY_URL=/api/__clerk` |
| DB connection errors | Use Neon **pooled** URL with `sslmode=require` |
| Stripe charges fail | Confirm live keys on Render; unset `PAYMENTS_MOCK_MODE` |
| Mobile can't reach API | `EXPO_PUBLIC_DOMAIN` must be `haulbrokr.com` (no `https://`) |
| Admin page denies access | Seed staff users + set `STAFF_AUTH_SECRET` on Render; login at `/admin/login` |
| Staff login loops / 401 | Same-origin cookies: use `haulbrokr.com` (not raw Render URL for admin UI) |
| Render cold starts | Starter plan spins down after idle — first request may be slow (~30s) |
